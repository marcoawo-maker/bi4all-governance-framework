# YAML Generation Proof of Concept

## Objective
Demonstrate that ingestion pipeline YAML can be generated dynamically from governance metadata stored in SQL and exposed through a Power Apps interface.

## Components
- Governance table: `admin.copyDataConfig`
- SQL views:
  - `admin.vw_copyDataConfig_Yaml`
  - `admin.vw_copyDataConfig_Yaml_Latest`
- Power Apps screen:
  - configuration dropdown
  - Generate YAML button
  - YAML preview box
  - Export YAML button
- Power Automate flow:
  - `ExportYamlFile`
- Output repository:
  - OneDrive folder for generated YAML files

## End-to-end flow
1. User selects an ingestion configuration in Power Apps.
2. Power Apps reads the YAML definition generated by SQL view `admin.vw_copyDataConfig_Yaml_Latest`.
3. The YAML is previewed in the application.
4. User exports the YAML through Power Automate.
5. Power Automate creates a `.yaml` file in OneDrive.

## SQL logic
The SQL layer provides:
- a full YAML generation view for all governance rows
- a filtered/latest YAML generation view to avoid duplicate historical versions in app consumption

## Power Apps logic
The YAML preview is generated from the selected configuration.

### Generate YAML button
```powerfx
Set(
    varYaml;
    LookUp(
        'admin.vw_copyDataConfig_Yaml_Latest';
        configId = Dropdown2.Selected.configId
    ).yaml_definition
)

ExportYamlFile.Run(varYaml; Dropdown2.Selected.destinationObjectPattern)
